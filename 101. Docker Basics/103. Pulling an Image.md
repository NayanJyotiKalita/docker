# Pulling an Image

## Scenario
You are a Release Engineer tasked with preparing a deployment artifact for a legacy web application. The application has been tested and certified to run specifically on Nginx version 1.23.4. Using a different version could lead to compatibility issues in production.

Your objective is to fetch this specific image from the official Docker registry. Additionally, your team's audit policy requires you to maintain a log of the download operation to prove that the correct artifact was retrieved.

## Task

Perform the following operations:

## 1. Image Retrieval
Pull the official Nginx image from the Docker registry.

Constraint: You must specify the tag 1.23.4. Do not pull the latest tag.

## 2. Audit Logging
Execute the pull command in a way that captures its output logs (both standard output and error messages).

Store this output in a file named: /home/user/answer.txt

## Expected Outcome
A local Docker image named nginx with the tag 1.23.4 must exist on the system.

The file /home/user/answer.txt must contain the logs generated during the pull process (e.g., "Downloaded newer image for…").

---

# COMMANDS

```shell
# Pull the specific version of Nginx and capture all output to the answer file
sudo docker pull nginx:1.23.4 2>&1 | tee /home/user/answer.txt

# Verify the image is present
docker images

# Verify the log file was created and contains content
cat /home/user/answer.txt
```
