# Docker Stop Container

## Scenario
You are a Database Administrator managing a development server. A junior developer has left a PostgreSQL database container running in the background, consuming valuable system resources. The project it was used for has been cancelled, so the database is no longer needed.

Your task is to perform a clean decommissioning of this service. You must safely stop the running container, remove the container instance to free up the name, and finally delete the underlying image to reclaim disk space.

#### Task
Perform the following cleanup operations in order:

### 1. Stop the Service
Identify the running container named my-postgres.
Gracefully stop this container to ensure the database shuts down correctly.

### 2. Remove the Container
Once stopped, remove the container instance my-postgres from the system.
Note: You cannot remove a running container without forcing it, so ensuring it is stopped first is a best practice.

### 3. Clean Up the Image
After the container is gone, the base image postgres is no longer required.
Remove the postgres image from the local cache to free up storage.

### Expected Outcome
The container my-postgres should no longer appear in docker ps -a.
The image postgres should no longer appear in docker images.
