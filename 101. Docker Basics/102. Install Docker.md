# Install Docker

## Problem Description:

## Scenario
You are a DevOps Engineer configuring a new build server for your team. This server will be responsible for building and deploying containerized applications. However, the server is currently a fresh instance with no container runtime installed. </br>

You need to provision this server by installing the Docker Engine. This involves securely configuring the software repositories, installing the community edition, and ensuring the background service is active and managing processes. </br>

**Important Note:** This task requires administrative privileges. If prompted for a password while running sudo commands, use your password set for your root account

## Task
#### Install Docker Community Edition (CE) and configure the service on the system.

 ## 1. Repository Setup
Before installing the engine, you must configure the system to trust Docker's software source:

```
Step 1: Update the local package index to ensure the system is aware of the latest upstream changes.
Step 2: Install the necessary prerequisite packages that allow the package manager (apt) to use a repository over HTTPS.
Step 3: Download and add Docker’s official GPG key to the system keyring. This is critical for verifying the integrity of the software.
Step 4: Add the stable Docker repository to the system's source list.
```

 ## 2. Installation & Service Management
Once the repository is configured:

```
Step 1: Install the docker-ce (Community Edition) package.
Step 2: Start the Docker daemon using the system's service manager.
Step 3: Enable the service to ensure it starts automatically if the server reboots.
Step 4: Verify the installation by checking the status of the service or printing the installed version.
```

## Expected Outcome
The Docker service must be in an active (running) state. </br>
The docker command-line tool must be accessible. </br>
The system must be able to return the Docker version without errors. </br>
